# IP Router

## 路由原则

用户明确指定 IP 时，优先使用指定 IP。用户没有指定时，先拆分文章配图点，再按 `shot-routing-rules.md` 为每张图选择一个最合适的 IP。

一篇文章可以使用多个 IP；一张图只能使用一个 IP。

## 内置 IP

### pageworker

适合：

- 文档、笔记、知识管理
- 信息整理、内容系统、素材归档
- 方法论梳理、文章结构、卡片化表达
- 从混乱内容到清晰结构

关键词：

- 文档、卡片、笔记、知识库、Notion、整理、归档、信息结构、内容系统、素材库、写作系统

默认理由模板：

这篇内容的核心是把信息整理成结构，适合用页工处理纸面工作流。

### threadsmith

适合：

- 路径、流程、漏斗、承接链路
- 用户旅程、转化路径、增长流程
- 断点修补、上下游连接、从 A 到 B 的移动关系
- 产品流程和协作链路

关键词：

- 路径、流程、漏斗、转化、承接、链路、断点、用户旅程、增长、接住、串联、闭环

默认理由模板：

这篇内容的核心是连接路径和修补断点，适合用线匠处理链路。

### holeseeker

适合：

- debug、问题诊断、复盘
- 风险、常见坑、故障、卡点
- 找根因、补漏洞、识别隐患
- 系统异常和流程失效

关键词：

- debug、bug、诊断、复盘、风险、坑、故障、漏洞、根因、异常、卡住、排查、失效

默认理由模板：

这篇内容的核心是发现问题和定位根因，适合用洞探进入问题现场。

## 文章级策略

- 如果用户要求“整篇统一 IP”，选择最贴近文章主线的 IP。
- 如果用户没有要求统一，默认逐图路由。
- 如果用户要求“对比不同 IP 效果”，为同一个观点分别输出不同 IP 的方案，但每张方案仍然只使用一个 IP。

## 冲突处理

- 同一个 shot 同时命中多个 IP 时，选择最贴近该 shot 核心动作的 IP，而不是关键词最多的 IP。
- 如果核心动作是“整理信息”，选 `pageworker`。
- 如果核心动作是“连接路径”，选 `threadsmith`。
- 如果核心动作是“发现问题”，选 `holeseeker`。
- 无法判断某个 shot 时默认 `pageworker`。
