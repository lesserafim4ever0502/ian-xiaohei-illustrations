# Shot Routing Rules

## 核心原则

逐图路由，而不是默认整篇文章只选一个 IP。

- 一篇文章可以使用多个 IP。
- 一张图只能使用一个 IP。
- 每个 shot 先判断“这张图的核心动作”，再选择 IP。
- 不按关键词数量投票，按画面里角色真正要做的事判断。

## 动作到 IP

### pageworker

选择条件：

- 这张图要表达整理、归档、压平、卡片化、结构化。
- 画面核心是把混乱信息变成可读结构。
- 重点是文档、知识库、素材库、文章结构、内容系统。

动作词：

- 整理、归档、贴签、折叠、压平、卡片化、沉淀、收纳、分类。

### threadsmith

选择条件：

- 这张图要表达连接、承接、路径、转化、闭环、断点修补。
- 画面核心是把 A 接到 B，或修补流程中断的位置。
- 重点是用户路径、增长漏斗、协作流程、上下游链路。

动作词：

- 连接、缝合、承接、转化、闭环、拉直、剪枝、打结、接住。

### holeseeker

选择条件：

- 这张图要表达排查、诊断、复盘、风险、坑、根因。
- 画面核心是发现问题藏在哪里，或解释为什么失效。
- 重点是 debug、故障、漏洞、异常、失败经验。

动作词：

- 排查、照出、标记、测深、复盘、找根因、补漏洞、圈风险。

## 冲突判断

- 如果一个 shot 同时有“整理”和“诊断”，看角色是在整理材料还是找问题；整理选 `pageworker`，找问题选 `holeseeker`。
- 如果一个 shot 同时有“整理”和“连接”，看核心是结构化内容还是修补路径；结构化选 `pageworker`，路径修补选 `threadsmith`。
- 如果一个 shot 同时有“连接”和“诊断”，看核心是修链路还是查失败原因；修链路选 `threadsmith`，查原因选 `holeseeker`。
- 如果仍然不明确，默认 `pageworker`，并在 shot list 里说明“不确定时用默认信息整理 IP”。

## Shot List 输出字段

每个 shot 必须包含：

- 放在哪段后
- 选择 IP
- 选择理由
- 图的主题
- 核心意思
- 结构类型
- 当前 IP 在图里做什么
- 建议元素
- 建议中文标注词
